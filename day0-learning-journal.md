##  Deliverable 2: Feynman Check (Day 0)

### Concept: API 

An API is like a waiter in a restaurant.
You (the user) tell the waiter what you want (your request). The waiter takes that request to the kitchen (server), where the food is prepared. Then the waiter brings the food back to you (response).

If the system doesn't work properly for example, if the waiter gives the wrong order or the kitchen misunderstands you either get the wrong result or no result at all.

So, an API helps different systems communicate clearly by acting as a middle layer that carries requests and responses between them.
